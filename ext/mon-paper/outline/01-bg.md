## S1: Case Studies

### S1.1: Lessons from AMR

**Job**: Demonstrate diagnosis asymmetry, extract 3 lessons (setup + 3 lessons + conclusion = ~4-5 paragraphs).

- Setup: studied AMR codes, found multiple pathologies, months of iterative debugging
- Lesson 1: Fine-grained telemetry with application context necessary (profiling insufficient)
- Lesson 2: Existing tracers inadequate (rigid schemas, CSV/JSON not queryable)
- Lesson 3: OLAP analytics powerful but disconnected (Parquet 50X faster, ClickHouse works, but painful ETL)
- Conclusion: diagnosis asymmetry demonstrated (months vs days), lessons point toward opportunity

### S1.2: Parallel Evolution in ML Training

**Job**: Validate problem is fundamental to BSP (1 paragraph).

- PyTorch Kineto/HTA/Dynolog: same pattern (post-hoc analytics), independent convergence validates fundamentality

**Handoff**: "These case studies reveal common patterns. To understand why existing tools have not solved them, we examine the landscape."

## S2: Why Existing Tools Fall Short

**Job**: Show neither HPC nor cloud tools solve it (2-3 paragraphs).

- HPC tools: post-hoc latency gaps (Score-P, TAU offline analysis, data deluge)
- Cloud tools: wrong model for BSP (Dapper/OTEL request-based, wrong causality boundaries)
- Neither solves it: HPC understands domain but post-hoc, Cloud real-time but wrong model

**Handoff**: "Given these limitations, what would effective BSP observability look like?"

## S3: Our Goal: On-Fabric Copilot for Hero Runs

**Job**: Paint the vision, hook the reader (1 paragraph).

- Vision: cluster-scale programmable observability for the run in progress (eBPF analogy)
- Scope: not multitenant service, not archival - copilot for expensive production run
- Goal: always-on, steerable (real-time control), programmable (flexible schemas/analyses/collection)
- Interactive workflow with real-time feedback, proportional data collection, complete feedback loop
- Hook: "This paper presents ORCA..."
