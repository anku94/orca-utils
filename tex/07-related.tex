\parahead{ML Training Observability}
The proliferation of bespoke ML training observability
systems~\cite{\citeGroupML}---each
reimplementing subsets of collection, detection, and
actuation---validates the need for a general substrate.
Greyhound~\cite{Wu2025:GREYHOUND} is closest to ORCA, implementing a
complete fixed-function feedback loop. ORCA 
enables these workflows via composable operators over consistent snapshots of the application state.
Mycroft~\cite{Deng2025:Mycroft} notes the scalability limits of
centralized telemetry that ORCA's tree-based overlay avoids by design.
% A detailed comparison is provided in Appendix~\ref{sec:appdx}.

\parahead{Microservice Observability}
Fay~\cite{Erlin2011:Fay}, Snicket~\cite{Berg2021:Snicket}, and Pivot Tracing~\cite{Mace2015:Pivottracing} pioneered 
causality-preserving query-driven tracing.
ORCA extends this paradigm to BSP, adding consistent control, open columnar formats, and
an RDMA TBON.

\parahead{Computational Steering}
Computational steering---interactive control of running simulations---dates to the early
1990s~\cite{Mille1998:SCIRun,Geist1997:Cumulvs} and continues through modern in-situ visualization
frameworks~\cite{Ayach2015:Catalyst,Larse2022:Ascent}, but remains realized as fixed-function
pipelines over domain-specific data models.
ORCA generalizes this as a workload-agnostic framework using columnar formats and relational
operators, with asynchronous and timestep-consistent control.
ORCA's motivation is telemetry analytics, but its applicability to broader steering use cases
tracks Arrow's adoption in scientific computing.
Particle data already maps naturally to
Arrow~\cite{Liu2022:ProcessingParticleData,Ulmer2023:ExtendingComposableDatab}, and bridges to
array-based formats are emerging~\cite{2026:arrow-zarr,:DLPackProtocolApache}.
As these ecosystems converge, the same infrastructure subsumes workflows from observability to
in-situ visualization and indexing.
Several extensions are natural:

\begin{itemize}[leftmargin=*]
  \item In-situ shuffling for partitioning~\cite{Zheng2018:DeltaFS,Jain2024:CARP} composes with ORCA's TBON as an
        intra-tier operation, enabling a model closer to Spark~\cite{Zahar2010:Spark} or Dask~\cite{Rockl2015:Dask}, but in-situ.
  \item Open table formats~\cite{:Iceberg}---catalog layers over
        Parquet---fit naturally, as ORCA's output is already partitioned.
  \item SmartNICs~\cite{Ulmer2023:ExtendingComposableDatab,Liu2022:ProcessingParticleData}
        as offload targets for plan fragments, automatically scheduled by ORCA's query planner.
  \item Dynamic instrumentation via eBPF~\cite{:eBPF} and
        DynInst~\cite{Berna2011:DynInst}, extending visibility
        into lower runtime layers.
% \item Sketch-based telemetry reduction~\cite{Sriva2024:Circa} produces
%       mergeable summaries that compose naturally with operator pushdown.
\end{itemize}
